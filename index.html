GoibiboBusiness WebView Integration documentation¶

The purpose of this documentation is to help our B2B partners integrate a customized version of the goibibo website into their native mobile app to allow end-users access to pricing and travel options they would not be able to find elsewhere. A high-level overview of the end-user experience is visualized below. Please refer to the links near the bottom of this page for specific details about the different steps.

Note

Depending on requirements, not all options are available for every provider - feel free to contact us for details!

Name Description
open-travel User selects travel option
verify-user B2B-Partner associates user with unique identifier
open-webview Login user on Goibibo via WebView API call
show-travel Display travel options to User
select-travel User finalizes travel plans
initiate-pay User selects ‘Make Payment’ button
pay-begin Request amount deducted from user account
process-payment Process payment request
pay-confirm After payment is confirmed, begin booking
book-travel Finalize travel details with travel provider
show-ticket User sees ticket within WebView
perform-travel User goes on travel

Integration Options¶

  • Initial Login
    • Send ALL user details to Goibibo
    • Request Goibibo to fetch user details
  • Payment Customization
    • Goibibo displays payment screens
    • B2B-Partner displays payment screens
